"Understanding Health Level 7" - HL7

Picture from https://www.hl7.org/implement/standards/
  • Health level seven(HL7) is a set of international standard of administrative and transfer of medical, clinical data between different healthcare provider, which focus on the application layer in OSI model..
  • HL7 is developed by health Level seven internation. which is a non-profit ANSI-accredited standards development organization
  • HL7 has been adopted by over 50 countries and over 1,600 members around the world including healthcare providers, goverment stakholders and medical companies.
  • HL7 including a series of standards:
    • HL7 Version 2.x
      -- A messiaging standard developed in 1989 that aims to support workflows. HL7 V2.x uses non-XML encoding syntax base on segments and delimiters.
    • HL7 Version 3
      -- A messiaging standard that used since 2005 that aims to support All healthcare workflows. HL7 V3 uses XML encoding syntax base and has a O-O design principle.
      • HL7 CDA
        -- Cincal Document Architecture is based on HL7 Version 3 that aim to specify the encoding, structure exchange the clinical documents. HL7 CDA was jointly published with ISO as ISO/HL7 27932
      • HL7 CCD
        -- Continuty of Care Document is aim to exchange of medical summaries which used in US, based on HL7 CDA.
      • HL7 SPL
        -- Structured Product Labeling is based on HL7 Version 3 that aims to describ the published information that accompanies a medicine.
    • HL7 CCOW
      -- Clinical Context Oject Workgroup is a protocol aims to share user context in different application in real time at user interface.
    • Other standards
      -- including FHIR, SAIF, Arden syntax etc.
  • The goal for HL7 is create a common framework for clinical data transfer, storage. Therefore they can be used or shared between different healthcare provider without making a mess.
